📂 S/W 분류 체계

용도별 S/W 분류 개념 및 제품군 현황 [출처: 소프트웨어분류.pdf]

TYPE 01

For Sale & SaaS

외부 판매 및 서비스로 매출 실적을 내는 S/W입니다.

  • Package: Desktop 실행 단품 저작도구 (Copy 당 과금)
  • SaaS: Cloud/Web 기반 서비스 (라이선스/ID 당 과금)
TYPE 02

Solution for Project

현장 문제점 해결을 위한 프로젝트성 S/W 및 사내 맞춤형 패키지입니다.

  • Solution: 한맥 가족사 ERP, 시공관리, 유지관리 플랫폼 등
  • 특징: 현장 맞춤식 기능 및 특정 도메인 최적화
TYPE 03

Inhouse S/W

엔지니어가 용역 매출을 올리거나 설계에 사용하기 위해 직접 만든 도구입니다.

  • Service: 사내 공간정보 제공 서비스 등
  • Package: Desktop 기반 설계/분석 보조 저작도구
TYPE 04

Module & Engine

가족사 프로그램의 기반이 되거나 핵심 부품·구성요소가 되는 S/W입니다.

  • Module: 특정 단일 기능 수행 독립적 요소
  • Engine: 다수 모듈 결합 및 S/W 핵심 기능 구현

① 사일로(Silo) 구조 — 부서 중심 분산
분류 구조가 S/W·업무·프로젝트로 혼재되어 일관성이 결여되어 있습니다.
  • 성과물 중심: S/W명 분류로 관리에 적합하나 부서업무 혼재
  • 부서 중심: 개발 문서가 시계열로 분산되어 탐색 어려움
  • 프로젝트 중심: 과제 종료 후 기술자산 활용 비효율 발생
② 비정형 계층 구조 — 규칙 부재
레벨별 규칙 불일치로 시스템적 정렬 및 제어가 불가능한 상태입니다.
  • 명칭 혼용: A.기획 / 01.기획 / 기획문서 등 무질서한 명명
  • 관리 공백: 신규 카테고리 생성 시 기존 체계와 간섭 발생
  • 관행 지속: "완성 후 업로드" 관행으로 인한 실시간 공유 단절
③ 임직원 관점 — 사용자 경험 부재
시스템 관리자 위주 구성으로 실무자의 접근 장벽이 높습니다.
  • 클릭 과다: 정보 도달까지 불필요한 계층 진입 반복
  • 예측 불가: 타 부서 문서 위치 유추가 불가능한 구조
  • 맥락 단절: 내가 어떤 단계의 참고 자료를 봐야 할지 불분명
