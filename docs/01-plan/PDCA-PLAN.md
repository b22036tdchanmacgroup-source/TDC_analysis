## 1. 개요

### 1.1 목적

문서관리시스템(TDC) 개선 방안 프로젝트의 진행 과정을 체계적으로 기록하고,
프로젝트 담당자와 보고 대상 임원이 관련 자료를 즉시 열람할 수 있는 웹 대시보드를 구축한다.

### 1.2 배경

- 프로젝트 관련 자료(보고서, 분석 결과, 회의록 등)가 분산되어 관리 비효율 발생
- 임원 보고 시 자료 취합 및 현황 파악에 소요 시간 과다
- PDF 기반 문서의 즉시 열람 환경 부재

### 1.3 관련 문서

- 프로젝트 청사진: `.hmcmkit/blueprint.md`
- 이전 대시보드 이력: 대화 ID `b617bc8e`, `e42631d8` 등 참조

## 6. 아키텍처 결정

### 6.1 프로젝트 레벨

| Level | 특성 | 선택 |
|-------|------|:----:|
| **Starter** | 단순 구조, HTML/CSS/JS | ✅ |
| Dynamic | Feature 기반, BaaS 연동 | ☐ |
| Enterprise | 레이어 분리, 마이크로서비스 | ☐ |

### 6.2 기술 결정

| 결정 사항 | 선택지 | 선택 | 이유 |
|----------|--------|------|------|
| 프레임워크 | 없음 (순수 HTML) | HTML5 | Starter 레벨, 빠른 개발 |
| 스타일링 | Vanilla CSS | CSS3 + Custom Properties | 프레임워크 불필요, 직접 제어 |
| 스크립트 | Vanilla JS | ES6+ | 의존성 최소화 |
| PDF 뷰어 | iframe / embed / object | iframe (기본) | 브라우저 내장 PDF 뷰어 활용 |
| 아이콘 | SVG 인라인 / 이모지 | SVG + 이모지 | 외부 의존성 없음 |
| 폰트 | Google Fonts | Pretendard + Inter | 한/영 최적화 |

### 6.3 폴더 구조

```

├── index.html              # 메인 대시보드 페이지
```
